# 🍞 Sistema de Controle de Estoque - Padaria

## 📋 Requisitos Funcionais (RF)

Os requisitos funcionais são as funções que o sistema deve fazer.

* **RF01** - O sistema deve cadastrar produtos.
* **RF02** - O sistema deve registrar a categoria dos produtos.
* **RF03** - O sistema deve informar a data de validade dos produtos.
* **RF04** - O sistema deve mostrar a quantidade de produtos em estoque.
* **RF05** - O sistema deve permitir aumentar ou diminuir o estoque.
* **RF06** - O sistema deve avisar quando o estoque estiver baixo.
* **RF07** - O sistema deve permitir filtrar produtos por categoria.
* **RF08** - O sistema deve mostrar produtos vencidos ou próximos do vencimento.
* **RF09** - O sistema deve permitir editar produtos cadastrados.
* **RF10** - O sistema deve permitir excluir produtos.

---

## ⚙️ Requisitos Não Funcionais (RNF)

Os requisitos não funcionais mostram como o sistema deve ser desenvolvido.

* **RNF01** - O sistema deve ser feito em Java.
* **RNF02** - O sistema deve usar MySQL como banco de dados.
* **RNF03** - O projeto deve ser criado com Maven.
* **RNF04** - O sistema deve funcionar no IntelliJ IDEA.
* **RNF05** - O sistema deve usar conexão com banco de dados usando JDBC.
* **RNF06** - O código deve ser organizado em pacotes (`model`, `dao` e `conexao`).
* **RNF07** - O sistema deve tratar erros usando `try-catch`.
* **RNF08** - O projeto deve ser salvo em um repositório no GitHub.
* **RNF09** - O sistema deve ser simples e fácil de usar.
* **RNF10** - O sistema deve responder rapidamente às consultas.
